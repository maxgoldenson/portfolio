ESP32 High-Speed Data Logger

A custom embedded system that continuously logs over 10,000 sensor samples per second for days — built to debug intermittent failures in real hardware that no off-the-shelf tool could capture.

01 — Problem

Intermittent failures that disappear when you look

Debugging real hardware mechanisms often means chasing failures that happen once every few hundred cycles — under load, at temperature, after hours of operation. A bench oscilloscope or serial monitor can't stay connected for days, and commercial data loggers either lack the sample rate, the right sensor interfaces, or cost a fortune.

At Brewbird, mechanism wear and control edge cases were creating failures that were difficult to reproduce in short test windows. I needed a tool that could live inside a machine, log continuously at high rate, and not get in the way.

"The failure only happens after 400 cycles. We need something that can watch the whole thing."

02 — Process

Architecting for speed and endurance

The key insight was that writing data to flash or SD synchronously at 10k samples/sec is too slow — the bottleneck is the storage medium, not the processor. The solution is to decouple acquisition from storage using a ring buffer in RAM.

01

Dual-core task architecture

Assigned Core 0 entirely to sensor polling at fixed intervals using a high-priority FreeRTOS task. Core 1 handles writing buffered data to storage. The two cores never block each other.

02

Ring buffer memory management

Implemented a lockless ring buffer in SRAM. When the write task falls behind, the buffer absorbs the burst without dropping samples. Buffer depth was sized for worst-case write latency plus margin.

03

Timestamping & data integrity

Each sample carries a 64-bit microsecond timestamp from the ESP32's hardware timer. CRC checksums on each flash page allow post-hoc detection of any write corruption.

04

Python post-processing

Wrote a Python script to pull logs over serial or WiFi, decode binary frames, and plot time series with annotated events. Made it easy to hand logs to teammates who aren't firmware engineers.

03 — Solution

A pocket-sized test instrument

The final logger fits on a single ESP32 dev board with a MicroSD breakout, runs from a USB power bank for multi-day deployments, and can be dropped into a running machine in minutes. Firmware is configurable — log rate, sensor channels, and trigger conditions are set via a simple config file on the SD card.

The tool has become a standard part of the hardware debug workflow — faster to deploy than setting up a DAQ, cheaper than any commercial equivalent, and purpose-built for the sensor types in use.
