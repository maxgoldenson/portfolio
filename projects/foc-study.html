FOC Motor Control Study

A first-principles study of Field-Oriented Control — designing experiments, writing firmware, and building deep intuition for Park-Clarke transforms, feedback loops, and BLDC drivers from the ground up.

01 — Problem

Using motor controllers without understanding them

Robotics projects routinely rely on off-the-shelf ESCs and motor drivers — but treating them as black boxes limits your ability to tune them, debug them, or design systems that push their limits.

I wanted to understand FOC deeply enough to implement it myself — starting from first principles: what is a rotating magnetic field, how does a three-phase winding produce it, and how do you control torque independent of speed?

"Reading a library's source code teaches you more about motor control than any tutorial. Writing your own teaches you everything else."

02 — Process

Experiment-driven learning

01

Clarke & Park transform derivation

Derived the Clarke (3-phase → 2-axis stationary) and Park (stationary → rotating reference frame) transforms from first principles. Verified numerically in Python before implementing in firmware.

02

Open-loop SVPWM implementation

Implemented Space Vector PWM on a microcontroller driving a 3-phase gate driver. Verified correct phase relationships on an oscilloscope before closing any loops.

03

Rotor position sensing

Integrated a magnetic encoder for rotor angle. Validated angle reading against known positions — the feedback that makes field orientation possible.

04

Current control loops (Id / Iq)

Implemented d-axis and q-axis PI controllers. Tuned gains empirically, observing torque response and current waveforms. Experimented with flux weakening for extended speed range.

03 — Solution

A working FOC implementation — and deep intuition

The outcome is foundational knowledge: a working FOC firmware stack, validated experiments, and the ability to reason from first principles about any motor control problem.

This directly improved my ability to specify, tune, and debug motor systems at Brewbird — particularly the grinder motor where torque consistency affects taste.

Understanding FOC at this level changes how you approach every motor selection, driver spec, and tuning conversation. It can't come from reading documentation — it requires building, breaking, and rebuilding until the physics is intuitive.

05 — Key Learning

Implementation is faster than documentation

Reading about the Park-Clarke transform gives you a formula. Implementing it, getting the wrong waveforms, and debugging back to the math gives you intuition that survives contact with a real motor. Tuning the PI current loops empirically — watching current waveforms respond to gain changes — taught more about control bandwidth in an afternoon than any textbook chapter.

The practical outcome: understanding FOC deeply enough to reason about tuning, diagnose problems, and specify drivers from first principles — not from copying settings from application notes. That level of understanding only comes from building it yourself.
